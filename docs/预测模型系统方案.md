# SIPLACE SX2 预测模型系统方案

## 概述

本方案详细说明如何**喂数据给模型**以及**如何标注数据**。

---

## 一、数据标注流程

### 1.1 数据来源

数据通过以下方式进入系统：

```
OPC UA接口 → SIPLACE Software → device_parameters表 (1Hz采集频率)
```

**采集的核心参数：**

| 参数名称 | 说明 | 频率 |
|---------|------|------|
| Nozzle_Vacuum_Level | 吸嘴真空度 | 每拾取周期 |
| Pick_Error_Flag | 拾取错误标志 | 每拾取周期 |
| Main_Air_Pressure | 主气路压力 | 每秒 |
| Vacuum_Generator_Pressure | 真空发生器压力 | 每秒 |
| Nozzle_Pick_Count | 吸嘴拾取次数 | 每日 |

### 1.2 标注数据的入口

#### 方式1：从预警确认标注

1. 在"预警管理"页面查看AI预警
2. 确认预警是否准确
3. 系统自动创建标注数据

#### 方式2：从维修工单标注

1. 在"工单管理"页面完成维修
2. 记录实际故障情况
3. 关联对应时间段的数据

#### 方式3：手动标注（推荐）

进入"数据管理"页面：

1. **选择时间窗口**
   - 选择设备
   - 设置开始时间和结束时间（建议4小时窗口）
   - 系统会加载该时间段的参数曲线

2. **标注标签**
   ```typescript
   {
     label_type: "nozzle_clog",      // 吸嘴堵塞
     is_fault: true,                 // 是否故障
     fault_severity: "moderate",    // 严重程度
     fault_time: "2026-03-12T10:30:00Z", // 故障发生时间
     train_split: "train",           // 训练集/验证集/测试集
   }
   ```

3. **特征提取**
   系统自动计算以下特征：
   - 统计特征：均值、标准差、斜率、最小值、最大值
   - 时域特征：峰值计数、过零点次数
   - 领域特征：真空度衰减率、抛料率变化

## 二、数据标注规范

### 2.1 标签类型定义

| 标签代码 | 中文名称 | 说明 |
|---------|---------|------|
| `nozzle_clog` | 吸嘴堵塞 | 真空度下降，抛料率上升 |
| `pressure_failure` | 气压故障 | 主气路压力异常 |
| `feeder_jam` | 送料器卡料 | 送料器故障 |
| `normal_operation` | 正常运行 | 设备正常工作 |

### 2.2 时间窗口规范

**推荐窗口大小：4小时（240分钟）**

原因：
- LSTM模型输入形状为 `(240, 20)` - 240分钟数据
- 包含足够的上下文信息
- 计算效率适中

**故障标注位置：**
```
[训练数据窗口 4小时] [故障发生点]
                      ↑
               标注fault_time在此处
```

### 2.3 数据集划分

| 数据集 | 比例 | 用途 |
|--------|------|------|
| 训练集 (train) | 70% | 模型训练 |
| 验证集 (val) | 15% | 超参数调优 |
| 测试集 (test) | 15% | 最终评估 |

**划分原则：**
- 按时间划分，避免数据泄露
- 保证每个数据集都有正负样本
- 故障样本尽量均匀分布

## 三、模型训练流程

### 3.1 开始训练

进入"模型训练"页面：

1. **配置训练任务**
   ```typescript
   {
     model_type: "lstm",           // 模型类型
     target_label_type: "nozzle_clog", // 预测目标
     config: {
       epochs: 50,                // 训练轮数
       batch_size: 32,             // 批次大小
       learning_rate: 0.001,       // 学习率
     },
     hyperparameters: {
       lstm_units: [64, 32],       // LSTM单元数
       dropout: 0.2,               // Dropout率
     }
   }
   ```

2. **点击"开始训练"**
   - 系统自动从`labeled_data`表加载数据
   - 提取特征并构建训练样本
   - 开始训练过程

### 3.2 训练数据统计

训练时系统会显示：
- 总样本数
- 故障样本数
- 正常样本数
- 训练/验证/测试集划分

### 3.3 评估指标

训练完成后查看：

| 指标 | 说明 | 目标值 |
|------|------|--------|
| Precision (精确率) | 预测为故障中真正故障的比例 | > 85% |
| Recall (召回率) | 真实故障中被正确预测的比例 | > 80% |
| F1 Score | 精确率和召回率的调和平均 | > 82% |
| Accuracy (准确率) | 整体预测正确的比例 | > 88% |

## 四、模型部署流程

### 4.1 选择模型部署

1. 在"模型训练"页面查看已完成的训练
2. 对比不同版本的评估指标
3. 选择效果最好的模型点击"部署"

### 4.2 部署后的效果

部署后，系统会：
- 使用该模型进行实时预测
- 在"预警管理"页面显示预测结果
- 记录预测历史到`prediction_history`表

### 4.3 模型版本管理

- 支持多个版本并存
- 同类型模型只有一个处于"已部署"状态
- 可以随时切换部署版本

## 五、闭环学习流程

```
1. 设备运行 → 采集数据 (device_parameters)
   ↓
2. 模型预测 → 生成预警 (alerts)
   ↓
3. 工程师处理 → 确认/解决
   ↓
4. 标注数据 → 标记是否为真故障 (labeled_data)
   ↓
5. 重新训练 → 优化模型 (model_trainings)
   ↓
6. 部署新版本 → 回到步骤1
```

## 六、快速开始指南

### 第一步：准备数据

1. 确保设备参数已采集到`device_parameters`表
2. 至少有几条维修记录或故障事件

### 第二步：创建标注

1. 进入"数据管理"页面
2. 点击"创建标注"
3. 选择包含故障的时间窗口
4. 填写标签信息

目标：先创建 **50-100条** 标注数据（正负样本都要有）

### 第三步：训练模型

1. 进入"模型训练"页面
2. 点击"开始训练"
3. 选择模型类型和预测目标
4. 等待训练完成

### 第四步：部署验证

1. 查看评估指标
2. 部署效果好的模型
3. 在"预警管理"页面验证预测效果

### 第五步：持续迭代

1. 收集更多标注数据
2. 定期重新训练
3. 监控预测准确率

## 七、API接口参考

### 标注数据管理

```typescript
// 获取标注数据列表
GET /api/labeled-data?label_type=nozzle_clog&is_fault=true

// 创建标注数据
POST /api/labeled-data
{
  device_id: "uuid",
  start_time: "2026-03-12T08:00:00Z",
  end_time: "2026-03-12T12:00:00Z",
  label_type: "nozzle_clog",
  is_fault: true,
  train_split: "train"
}

// 获取单条标注详情（包含参数数据）
GET /api/labeled-data/[id]
```

### 模型训练管理

```typescript
// 创建训练任务
POST /api/model-trainings
{
  model_type: "lstm",
  target_label_type: "nozzle_clog",
  config: { epochs: 50, batch_size: 32 }
}

// 部署模型
POST /api/model-trainings/[id]/deploy

// 获取训练记录
GET /api/model-trainings?is_deployed=true
```

## 八、数据标注最佳实践

### ✅ 推荐做法

1. **标注完整故障周期**
   - 从故障发生前2小时开始
   - 到故障处理后1小时结束
   - 包含完整的变化过程

2. **保持样本平衡**
   - 正负样本比例尽量接近 1:1
   - 最少 30-50 个故障样本
   - 最少 30-50 个正常样本

3. **高质量标注**
   - 准确记录故障发生时间
   - 详细描述故障现象
   - 注明故障严重程度

4. **多样化场景**
   - 不同设备的数据
   - 不同时间段的数据
   - 不同严重程度的故障

### ❌ 避免的做法

1. 不要只标注明显故障
2. 不要标注时间太短的窗口
3. 不要所有数据都放在训练集
4. 不要频繁删除和重新标注

## 总结

通过这套系统，你可以：

1. **标注数据** - 通过"数据管理"页面创建高质量标注
2. **训练模型** - 通过"模型训练"页面训练LSTM等模型
3. **部署验证** - 部署模型并验证预测效果
4. **持续优化** - 闭环学习，不断提升准确率

开始使用吧！
