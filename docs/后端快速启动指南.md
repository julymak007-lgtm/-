## 🚀 快速启动（3步）

### 方式1：全栈一键启动（推荐）

```bash
# 在项目根目录执行
./scripts/start-fullstack.sh
```

这个脚本会：
- 自动安装Python依赖
- 启动Python后端（端口8000）
- 启动Next.js前端（端口5000）

---

### 方式2：仅启动后端

```bash
# 进入后端目录
cd backend

# 安装依赖（首次）
./scripts/install.sh

# 启动后端
./scripts/start.sh
```

## 📋 详细步骤

### 1. 配置环境变量

```bash
cd backend

# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件，配置数据库连接
# 填入你的Supabase数据库信息
```

`.env` 文件配置示例：

```env
# FastAPI 服务
BACKEND_HOST=0.0.0.0
BACKEND_PORT=8000
BACKEND_RELOAD=true

# 数据库连接（从Supabase获取）
SUPABASE_HOST=db.xxxxx.supabase.co
SUPABASE_PORT=5432
SUPABASE_DB=postgres
SUPABASE_USER=postgres
SUPABASE_PASSWORD=your_password

# 启用真实后端训练
USE_REAL_TRAINING=true
```

### 2. 启用真实后端模式

在Next.js中启用真实后端，编辑文件：

`src/app/api/model-trainings/route.ts`

```typescript
// 将这行改为 true
const USE_REAL_BACKEND = true;
```

或者在项目根目录的 `.env` 文件中添加：

```env
USE_REAL_BACKEND=true
BACKEND_URL=http://localhost:8000
```

### 3. 启动服务

#### 选项A：全栈启动（推荐）

```bash
# 在项目根目录
./scripts/start-fullstack.sh
```

#### 选项B：分别启动

**终端1 - 启动后端：**
```bash
cd backend
./scripts/install.sh  # 首次需要
./scripts/start.sh
```

**终端2 - 启动前端：**
```bash
# 在项目根目录
pnpm dev
```

## 📊 模型功能

### LSTM模型

- **用途**: 故障分类预测
- **架构**: 2层LSTM（64+32单元）
- **输入**: 240分钟时间窗口 × 20个特征
- **输出**: 故障概率（0-1）

### 生存分析模型

- **用途**: 剩余使用寿命(RUL)预测
- **支持模型**: Weibull AFT, Cox PH, 随机生存森林
- **输出**: 生存函数 + 预期寿命

## 🐛 常见问题

### Q: Python版本要求？

A: Python 3.9+，推荐 3.10 或 3.11

### Q: 后端启动失败？

A: 检查以下几点：
1. 端口8000是否被占用
2. .env文件配置是否正确
3. Python依赖是否安装成功

### Q: 如何查看后端日志？

A: 
```bash
# 如果使用全栈启动脚本
tail -f backend/logs/backend.log

# 或直接查看后端输出
cd backend && ./scripts/start.sh
```

### Q: 训练速度很慢？

A: 
- LSTM训练需要较多计算资源
- 可以减小batch_size或epochs
- 有条件可以使用GPU加速

### Q: 想只用前端演示？

A: 没问题！保持 `USE_REAL_BACKEND = false` 即可，不需要启动Python后端。
